# ナレッジ・コンテキスト・申し送り

実際の使用で得た知見をここに蓄積する。
テンプレートや CLAUDE.md への反映候補もここに記録する。

---

## AI ごとの特性（実地知見）

### Web Gemini（PM役・設計役）

**良かった点:**

- テンプレートのおかげで「整理モード」が機能した
- 推測で突き進まず、質問してくれるようになった
- Gem 化すると毎回コピペ不要で便利

**課題:**

- 添付資料（Google ドキュメント・NotebookLM・GitHub リポジトリ）を読んでくれないことがある
  → 対策：NotebookLM で事前にリサーチして、その結果テキストを直接貼り付ける
- 長い会話でパンクする
  → 対策：PM役と設計役を必ず別チャットに分ける

### Claude Code（実装役）

**良かった点:**

- 「任せて安心」という感覚
- 指示の意図をよく理解する
- GitHub Issues との連携が強い

**課題:**

- レート上限（5時間枠）にすぐ達する。2セッションで計6〜7時間弱
- コンテキストが溜まると上限到達が早くなる
- 初期プランが詳細で良いのにファイルに残らない（AAR より精緻）

**対策:**

- 初期プランを必ず `handoff/plan_YYYYMMDD.md` に保存させる
- タスクを小さく分割してセッション単位のコンテキスト量を抑える

### GitHub Copilot（補完役）

**良かった点:**

- Claude Code のテンプレートを使い回せた
- 細かい補完に強い

**課題:**

- 要件理解が弱い。Claude Code の初期プランがあると大幅に改善

**対策:**

- `handoff/plan_YYYYMMDD.md` を Copilot に見せてから作業させる
- Claude Code レート上限時のフォールバックとして活用

## ワークフロー上の発見

### リサーチャー役の有効性

NotebookLM + DeepResearch で事前にソースを追加し仕様を洗い出した後、
Gemini PM役に渡すフローが有効だった。

```text
NotebookLM（リサーチ） → 仕様メモ → Gemini（PM役） → spec.md
```

つまずきポイントや実現可能性を事前にインプットできる。
→ リサーチャー役テンプレートとして整備する価値あり

### handoff/ はコミットすべき

gitignore すると知見が失われる。

- 作業ログとして価値がある
- チームで共有すると有用
- AAR から改善事項を吸い上げ、テンプレートに反映するフィードバックループに使える

会社では `handoff/` をリポジトリに含めて運用。うまく機能した。

### 初期プランを必ず保存する

Claude Code がセッション開始時に出す「状況把握レポート＋作業計画」が最も詳細。
AAR より精緻なのでここを失うのはもったいない。

**対応:** `handoff/plan_YYYYMMDD.md` として保存を CLAUDE.md に明記する。

### CLAUDE.md の命名問題

チーム開発リポジトリでは `CLAUDE.md` をルートに置きにくい。

- 会社では `NOTE.md` という名前にして運用（Claude Code は任意ファイルを読める）
- セッション開始時に `「NOTE.md を読んでルールを把握して」` と指示

### ブランチ・コミット規約

AI がブランチを切らずに main 直接作業するリスクがある。
プロジェクトごとにコミット規約・ブランチ規約が異なる。

**対応:** CLAUDE.md にブランチ作成を必須化し、規約不明時は確認させる。

### 工数見積もりの価値

自分の工数見積もりが外れやすいので、AI に釘を刺してほしい。
全フェーズでの見積もりと、実装中の残工数更新があると助かる。

## 未解決の課題

### ウォーターフォール問題

PM → 設計 → アーキテクチャ → 実装 が全部直列で Claude Code の負荷が大きい。

**仮説:** アーキテクチャ検討・リスク見積もりが不十分で後から手戻りが発生している。

**検討中の対策:**

- 設計フェーズにリスク見積もりを必ず含める
- アーキテクチャをフェーズとして独立させる（今は設計役に混ざっている）
- タスクをより細粒度に分割してセッション完結できるサイズにする

### Claude Code レート上限時のフォールバック

上限到達 → Copilot でしのぐ際に、初期プラン（`handoff/plan_YYYYMMDD.md`）があれば
Copilot でも作業継続できる可能性がある。要検証。

### Gemini の添付資料読み込み問題

Google ドキュメント・NotebookLM・GitHub リポジトリを添付しても読まないことがある。
テンプレートで質問を促すことで「読まずに推測」は減ったが根本解決ではない。

現状のワークアラウンド: NotebookLM でリサーチした内容をテキストで直接貼り付ける。
