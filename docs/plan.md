## フェーズ 2：フロー改善（中期）

### ウォーターフォール問題への対応

現状の課題: PM → 設計 → 実装 がすべて順番待ちで Claude Code の負荷が高い。

改善方針（検討中）:
- タスクをより小さく分割してセッションサイズを抑える
- アーキテクチャフェーズを独立させる（Claude Code Architect セッション）
- リスク見積もりを設計フェーズに組み込む
- Claude Code のサブエージェントを使って並列実装する

### handoff/ の活用

- handoff/ をコミットして知見のログとして活用
- AAR から改善事項を定期的に吸い上げてテンプレートに反映
- このフィードバックループ自体をフローに組み込む

### AI 間連携の自動化

- Level 0（現在）: 人間がコピペ
- Level 1: スクリプトで Gemini CLI を呼び出す
- Level 2: API 統合で半自動化
