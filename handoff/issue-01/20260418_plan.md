# プラン：構造整備 + コンテンツ整合性チェックテンプレート

## Context

3つの問題を同時に解決する:
1. CLAUDE.md が NOTE.md を参照しているが、このリポジトリはAIチームワークフローを「使う」場所ではなく「育てる」場所なので不適切
2. このリポジトリ自身の作業ファイル（plan/aar）の置き場がない
3. 「ファイルはあるのに説明がない」「docs と実態がズレている」を検知する仕組みがない

静的 linter（GitHub Actions）は **入れない**。ユーザーが求めているのは AI によるセマンティックな整合性チェックであり、markdownlint が検知できる構文エラーではないため。

---

## タスク一覧

### TASK-1: CLAUDE.md を NOTE.md 参照から独立させる

**変更対象**: `CLAUDE.md`

- `See @ai-team/NOTE.md for detailed working rules.` の行を削除
- このリポジトリ固有の作業ルールを直接 CLAUDE.md に追記:
  - 作業前にブランチを切る
  - コミット・プッシュは確認してから
  - 作業 handoff は `handoff/issue-XX/` に保存する
  - セッション開始時のプランは `handoff/issue-XX/YYYYMMDD_plan.md` に保存する
- 構造図に root `handoff/` を追加

### TASK-2: ルート `handoff/` の新設と移動

**新設ディレクトリ**: `handoff/`

```text
handoff/                    ← このプロジェクト自身の作業ファイル
  issue-01/                 ← Issue #1（AIコードレビュー）の作業
    20260418_plan.md        ← 今回のプランをここにコピー保存
    .gitkeep
  misc/                     ← Issue に紐づかない作業・サンプル
    20260417_prototype_checkpoint.md  ← ai-team/handoff/review/ から移動
```

`ai-team/handoff/review/` を削除し、中身を `handoff/misc/` に移動。
`ai-team/handoff/` には `.gitkeep` のみ残す（テンプレート用空ディレクトリ）。

### TASK-3: 整合性チェックテンプレート新規作成

**新規ファイル**: `ai-team/templates/consistency_check.md`

以下の観点でチェックするテンプレート:

- **未記載ファイル検知**: ディレクトリにあるがどの README にも説明がないファイル/フォルダ
- **docs ↔ 実態の乖離**: docs/ に書いてある状況が実際のファイル構成と違う箇所
- **ファイル間矛盾**: 同じ概念について異なる説明が複数ファイルにある箇所
- **リンク切れ**: markdown 内のリンクが指すファイル/フォルダが実在しない箇所

使用シーン: VSCode 内 Claude Code で「整合性チェックして」 / PR 時にレビューとして使用。

### TASK-4: ai-team/README.md の構成図を更新

- `handoff/review/` の記述を削除
- `ai-team/handoff/` は「テンプレート用空ディレクトリ（他リポジトリにコピーして使う）」と明記

### TASK-5: Issue #1 にコメントで見積もりを投稿

各イシューに概算見積もりをコメント追加:

| Issue | 見積もり |

## 実装順序

0. ブランチ作成: `git checkout -b feature/structure-and-consistency-check`
1. TASK-2（handoff/新設・移動）→ ディレクトリ構造を先に確定
2. TASK-1（CLAUDE.md修正）→ 構造確定後に記述更新
3. TASK-3（整合性チェックテンプレート）→ 本日のメイン成果物
4. TASK-4（README更新）
5. TASK-5（Issueコメント）→ 最後にまとめて投稿
6. PR作成 → ユーザー（hirobirofran）にレビュー依頼

## 検証方法

- 整合性チェックテンプレートを Claude Code に貼り付けて、このリポジトリ自身に対して実行し、実際に問題を検出できるか確認
- CLAUDE.md から NOTE.md 参照を外した後、Claude Code が NOTE.md のルール（PM役・設計役フェーズ等）を勝手に参照しないことを確認
