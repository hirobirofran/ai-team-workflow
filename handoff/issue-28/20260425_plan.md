### 1.4 セッション分離の運用ルール（再確認）

- このセッションは **本プランの作成と本作業の実行** が範囲
- 別セッション Opus に **レビュー依頼** する場合は本ファイルと initial_prompt.md を渡す（コンテキストウィンドウ独立）
- 迷走したら手元巻き戻しではなく checkpoint→新セッション（`docs/knowledge.md`「セッション分離の一般化」）

### 1.5 library/ の性質（信頼性の最終防衛線）

`library/` は単なる情報置き場ではなく、**将来の AI / 人間が一次情報の真偽を確かめにくる最後の砦** という位置づけ。`docs/knowledge.md` に記録された Prisma Python クライアント実例（DR では推奨されるが GitHub ではアーカイブ済）は、library/ が機能していれば防げた事故の典型。

含意:

- エントリの正確性 = library/ そのものの信頼性。曖昧な要約や推測は載せない
- 一次情報 URL は **必ず付ける**（後追い検証の入口）
- 内容が陳腐化したら速やかに更新（古いまま残すと最終防衛線として機能しない）
