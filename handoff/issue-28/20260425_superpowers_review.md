### ⚠️ 解釈上の注意（前提崩れ⑦）

「Opus + Gemini と被らない新規指摘を 6 件出せた」という結果は、**プロンプト側で「両者と重複しない新規の指摘があれば優先的に出してください」と指示したため、被らない結果になるのは論理的に当然**。

→ 「被らない結果」自体を能力として評価するのは **バイアス**。

評価可能な本来の軸は: **「Opus と Gemini が実際に見逃していた論点を、subagent が拾い上げられたかどうか」**。指摘 6 件の内容を見ると、M-N2（self-apply 整合性）/ B-N3（レビュー多重化観点）/ O-N1（最新版運用と最終防衛線の矛盾）は **plan.md 本体と既存レビューの両方で扱われていなかった真の見逃し**であり、これは評価できる。

「被らない結果が出た」≠「能力が高い」。「実在する見逃しを拾った」=「能力が認められる」。混同しないこと。

### ⚠️ CWD ファイル参照の課題は同様

サブエージェントは Claude Code の Agent ツール経由なのでリポジトリ内のファイルは Read で取得できる（`plan_review.md` `gemini_review.md` を実際に読んでいた痕跡が出力にあり）。Gemini CLI と同じく **真の独立コンテキストではない**。

ただし「実装後レビュー」想定のサブエージェントは元々「コードベースを読んで判定する」前提なので、これは設計通り。**真の独立性が必要なケース**（前提崩れ自動検出など）にはこの設計では足りない可能性が残る。
