# Claude Code セッション開始テンプレート

> **使い方**: Claude Code を起動したら最初にこのプロンプトを貼り付ける。
> 「---ここから---」〜「---ここまで---」をコピーして使う。

---ここから---

## セッション開始

以下の手順で状況を把握してください。**実装はまだしないでください。**

### Step 1: リポジトリ構造の把握

- ファイルツリーを確認する
- 主要な設定ファイルを読む（terragrunt.hcl / package.json / etc.）

### Step 2: 成果物の確認

- `handoff/` ディレクトリがあれば内容を確認する
- 直近の AAR（`handoff/aar_*.md`）があれば読む
- 直近のプラン（`handoff/plan_*.md`）があれば読む

### Step 3: GitHub Issue の確認

```bash
gh issue list --state open
```

- 今日の作業に関係しそうな Issue を確認する

### Step 4: タスク仕様の確認

- `handoff/task_spec.md` があれば読む

### Step 5: 規約の確認

- コミットメッセージの規約を確認する（CONTRIBUTING.md / README / 既存コミット履歴）
- ブランチ命名規約を確認する（既存ブランチ一覧から推測、不明な場合は質問する）

### Step 6: 整理して報告

以下の形式で整理してください：

```text
## 状況把握レポート

### リポジトリ概要
（構造・技術スタック・特記事項）

### コミット・ブランチ規約
（確認できた規約。不明な場合は「確認できませんでした。以下で進めてよいですか？[案]」）

### 今日やるべき作業
（Issue番号・タスク名・優先度）

### 各タスクの影響ファイル
（タスクごとに列挙）

### 作業手順（案）
1. ブランチ作成: git checkout -b xxx
2. ...

### 工数見積もり
- 合計: 約XX時間
- タスク別: TASK-001: 約Xh / TASK-002: 約Xh

### ⚠️ 懸念点・確認事項
（不明な点・リスク）
```

整理が完了したら「この手順で進めてよいですか？」と確認してください。
