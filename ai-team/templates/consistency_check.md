# 整合性チェックテンプレート

> **使い方**: 変更作業後、またはPR作成前に実行する。
> VSCode 内 Claude Code への指示例：「consistency_check.md の手順でリポジトリの整合性をチェックして」
> PR 時にも使用可。GitHub Actions ではなく AI によるセマンティックチェック。

---ここから---

【整合性チェックモード】

## あなたの役割

このリポジトリ全体の整合性を監査します。**修正はしないでください。問題の報告だけ行います。**

## チェック項目

### 1. 未記載ファイル・フォルダ検知

リポジトリ内の実際のファイル構成を確認し、以下を調べる:

- `README.md` や `ai-team/README.md` のファイル構成図に載っていないファイル/フォルダ
- `CLAUDE.md` の構造図に載っていないファイル/フォルダ

```bash
# ファイル一覧取得（参考）
find . -not -path './.git/*' -not -name '.gitkeep' | sort
```

### 2. docs ↔ 実態の乖離

`docs/plan.md` の「現在の状態」「完了済み」チェックリストと実際のファイル状態を比較:

- ✅ になっているが実際には存在しない/未完成のもの
- 実際には完了しているが ✅ になっていないもの

### 3. ファイル間の矛盾

複数ファイルに同じ概念の説明があるとき、内容が食い違っていないか:

- フォルダ構成の説明（CLAUDE.md / README.md / ai-team/README.md / NOTE.md）
- handoff/ のパス表記（`handoff/issue-XX/` か `handoff/aar_*.md` か など）
- AI の役割・ツールの説明（README / templates / docs/）

### 4. リンク切れ

markdown 内の `[テキスト](パス)` リンクが指すファイル/フォルダが実在するか:

- 相対パスリンクの確認
- `@ファイル名` 参照（CLAUDE.md など）の確認

### 5. テンプレート ↔ 手順書の乖離

`ai-team/README.md` のステップ説明と `ai-team/templates/` の実際のテンプレートが対応しているか:

- README に記載のあるテンプレートがすべて存在するか
- テンプレートが存在するがREADMEに説明がないものはないか

## 出力フォーマット

```markdown
## 整合性チェック結果

**実行日**: YYYY-MM-DD
**対象リポジトリ**: （リポジトリ名）

### 🔴 問題あり（要対応）

| # | 種別 | 場所 | 内容 |
|---|------|------|------|
| 1 | 未記載ファイル | README.md | `handoff/misc/` の説明がない |
| 2 | ファイル間矛盾 | CLAUDE.md vs README.md | handoff パスの表記が異なる |

### 🟡 軽微（できれば対応）

| # | 種別 | 場所 | 内容 |
|---|------|------|------|
| 1 | docs乖離 | docs/plan.md | TASK-XX が ✅ だが未完成 |

### 🟢 問題なし

- （問題のなかったカテゴリ）

### 📋 修正候補サマリー

（優先度順に何を直すべきか）
```

整理が完了したら「この結果でよいですか？修正を始めてよいですか？」と確認してください。
